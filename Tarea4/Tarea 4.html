Tarea 4: Portada, búsquedas y páginas de detalle

Para esta tarea pondremos una portada a la tienda con funcionalidad de búsquedas, y paginas de detalle para cada producto. Dejamos para la tarea siguiente el carrito.

Seguimos el patrón de software MVC, con Prisma para el Model y plantillas Nunjucks para las Vistas.

La estructura, con carpetas separadas para el model, view y controler: 

├── prisma                   Model
⃑⃑│    ├── prisma.client.ts    Conexión BD
│    └── schema.prisma       Esquema
├── imagenes                 Imágenes
├── routes                   Controladores
│    └── productos.ts
├── views                    Vistas (html)
│    ├── base.njk
│    ├── portada.njk
│    └── detalle.njk
├── .env                     Variables de entorno (y passwords)
├── docker-compose.yml       Base De Datos PostgreSQL
├── index.ts                 Servidor
└── package.json

En el servidor index.ts: 

...
// middelware para servidor de archivos: url /public/imagenes servida desde carpeta imagenes
app.use('/public/imagenes', express.static('imagenes'))   

// 'último' middleware
import ProductosRouter from "./routes/productos.ts"
app.use('/', ProductosRouter)
en routes/productos.ts 
import express from "express"
import prisma from "../prisma/prisma.client.ts"
const router = express.Router()

// portada
router.get ('/', async (req, res) => {
  try {
    const cards = await prisma.producto.findMany({
      omit: {
        descripción:true
      }
    })
    res.render('portada.njk', {cards})
		
  } catch(error) {
    console.error("~ error:", error.message)
    res.status(500).send(`Error: ${error.message}`)
  }
})

// añadir otro para detalle en /producto/:id
// y otro para las búsquedas en /buscar?busqueda=Lámina 

export default router

Para las plantillas tendremos en cuenta la herencia, poniendo en archivo base.njk el esqueleto de html, con lo que no cambia de página a página.

El html y css podemos pedirselo a la IA, a partir de capturas de pantalla de Tienda Prado. Para el css usar Bootstrap.
