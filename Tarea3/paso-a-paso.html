🚀 Tarea 3: PostgreSQL con Prisma ORM

Guía completa paso a paso para principiantes

📖Introducción

En esta tarea aprenderás a configurar una base de datos PostgreSQL usando Docker y a interactuar con ella mediante Prisma ORM.

¿Qué es un ORM?

Un ORM (Object Relational Mapping) es una capa de software que te permite trabajar con bases de datos usando objetos de tu lenguaje de programación, en lugar de escribir SQL directamente. Prisma es uno de los ORMs más modernos y populares para Node.js.

¿Por qué Docker?

Docker nos permite ejecutar PostgreSQL en un contenedor aislado, sin necesidad de instalarlo directamente en nuestro sistema. Esto facilita la configuración y evita conflictos.

0Requisitos Previos

Antes de comenzar, asegúrate de tener instalado:

  • Node.js (versión 18 o superior) - Descargar aquí
  • Docker Desktop - Descargar aquí
  • Un editor de código (VS Code recomendado)
  • Una terminal (CMD, PowerShell, Terminal, etc.)
💡 Verificar instalación:
Abre una terminal y ejecuta:
node --version (debe mostrar v18 o superior)
docker --version (debe mostrar la versión de Docker)

1Crear el Proyecto

Primero, crea una carpeta para tu proyecto y navega a ella:

mkdir Tarea3
cd Tarea3

Inicializa un proyecto Node.js:

npm init -y
✅ Esto crea un archivo package.json que contendrá la información de tu proyecto.

2Instalar Dependencias

Instala las librerías necesarias para trabajar con Prisma y PostgreSQL:

npm install @prisma/client @prisma/adapter-pg
npm install -D prisma dotenv tsx

¿Qué instalamos?

  • @prisma/client: Cliente para interactuar con la base de datos
  • @prisma/adapter-pg: Adaptador específico para PostgreSQL
  • prisma: Herramienta CLI de Prisma (desarrollo)
  • dotenv: Para cargar variables de entorno desde archivo .env
  • tsx: Para ejecutar archivos TypeScript directamente
⚠️ Importante: Usamos Prisma 6 en lugar de Prisma 7 porque la versión 7 tiene problemas de compatibilidad. Asegúrate de que tu package.json especifique la versión 6.

3Configurar Variables de Entorno

Crea un archivo .env en la raíz del proyecto con el siguiente contenido:

.env
POSTGRES_USER=yo POSTGRES_PASSWORD=una_clave_muy_segura_123 POSTGRES_DB=ssbw DATABASE_URL="postgresql://yo:una_clave_muy_segura_123@localhost:5432/ssbw?schema=public"

Explicación de cada variable:

  • POSTGRES_USER: Nombre de usuario para PostgreSQL
  • POSTGRES_PASSWORD: Contraseña del usuario
  • POSTGRES_DB: Nombre de la base de datos que se creará
  • DATABASE_URL: URL completa de conexión que usa Prisma
🔒 Seguridad: El archivo .env contiene información sensible. NUNCA lo subas a repositorios públicos. Agrégalo a tu .gitignore.

4Configurar Docker Compose

Crea un archivo docker-compose.yml para configurar PostgreSQL:

docker-compose.yml
version: '3.8' services: db: image: postgres:alpine container_name: postgres_ssbw restart: always environment: POSTGRES_USER: ${POSTGRES_USER} POSTGRES_PASSWORD: ${POSTGRES_PASSWORD} POSTGRES_DB: ${POSTGRES_DB} ports: - "5432:5432" volumes: - postgres_data:/var/lib/postgresql/data volumes: postgres_data:

¿Qué hace este archivo?

  • Usa la imagen postgres:alpine (versión ligera de PostgreSQL)
  • Nombra el contenedor como postgres_ssbw
  • Mapea el puerto 5432 (puerto estándar de PostgreSQL)
  • Crea un volumen para persistir los datos
  • Lee las credenciales desde el archivo .env

5Iniciar PostgreSQL

Inicia el contenedor de PostgreSQL con Docker Compose:

docker compose up -d

¿Qué hace este comando?

  • docker compose up: Inicia los servicios definidos en docker-compose.yml
  • -d: Ejecuta en segundo plano (detached mode)

Verifica que el contenedor está corriendo:

docker ps
✅ Deberías ver un contenedor llamado postgres_ssbw en estado "Up"

6Configurar el Schema de Prisma

Crea la carpeta prisma y dentro el archivo schema.prisma:

prisma/schema.prisma
generator client { provider = "prisma-client-js" output = "../generated/prisma" previewFeatures = ["fullTextSearchPostgres"] } datasource db { provider = "postgresql" url = env("DATABASE_URL") } model Producto { id Int @id @default(autoincrement()) título String @db.Char(127) descripción String @db.Text precio Decimal imagen String @db.Char(127) }

Explicación del schema:

  • generator: Configura cómo se generará el cliente de Prisma
  • datasource: Define la conexión a PostgreSQL
  • model Producto: Define la estructura de la tabla de productos
  • @id: Marca el campo como clave primaria
  • @default(autoincrement()): El ID se genera automáticamente
  • Decimal: Tipo de dato preciso para precios (mejor que Float)

7Crear la Migración

Una migración es un archivo que contiene los cambios en la estructura de la base de datos:

npx prisma migrate dev --name esquema_inicial

¿Qué hace este comando?

  • Lee el archivo schema.prisma
  • Genera el SQL necesario para crear las tablas
  • Aplica los cambios en PostgreSQL
  • Guarda la migración en prisma/migrations/
  • Genera automáticamente el cliente de Prisma
✅ Si todo va bien, verás el mensaje "Your database is now in sync with your schema"

8Configurar el Cliente de Prisma

Crea el archivo prisma/prisma.client.ts:

prisma/prisma.client.ts
import 'dotenv/config'; import { PrismaClient } from '../generated/prisma/index.js' import { PrismaPg } from '@prisma/adapter-pg' const adapter = new PrismaPg({ connectionString: process.env.DATABASE_URL }) const prisma = new PrismaClient({ adapter }) export default prisma

¿Qué hace este archivo?

  • Carga las variables de entorno del archivo .env
  • Configura el adaptador de PostgreSQL
  • Crea una instancia de PrismaClient
  • Exporta la instancia para usarla en otros archivos

9Crear Script para Poblar la Base de Datos

Crea el archivo seed.ts para insertar productos en la base de datos:

seed.ts
import prisma from "./prisma/prisma.client.ts"; import productos from "./productos.json" with {type: 'json'} await Guardar_en_DB(productos) // Consultas de ejemplo const total = await prisma.producto.count() console.log(`Total de productos: ${total}`) await prisma.$disconnect() async function Guardar_en_DB(productos) { for (const producto of productos) { const precio = Number( producto.texto_precio.slice(0, -2).replace(/,/, '.') ) await prisma.producto.create({ data: { título: producto.título, descripción: producto.descripción, imagen: producto.imagen, precio } }) } }

Agrega el script al package.json:

"scripts": { "seed": "tsx seed.ts", "consulta": "tsx consulta.ts" }

10Poblar la Base de Datos

Ejecuta el script de seed para insertar los productos:

npm run seed
✅ Verás mensajes de "Creado:" para cada producto insertado, seguido de estadísticas de la base de datos.

12Explorar con Prisma Studio

Prisma Studio es una interfaz gráfica para ver y editar tus datos:

npx prisma studio

Esto abrirá una página web en http://localhost:5555 donde podrás:

  • Ver todos los productos
  • Editar registros
  • Eliminar datos
  • Crear nuevos registros
💡 Tip: Prisma Studio es muy útil para debugging y desarrollo.

🔧Solución de Problemas

Error: "Can't reach database server"

  • Verifica que Docker esté corriendo: docker ps
  • Verifica que el puerto 5432 no esté ocupado
  • Reinicia el contenedor: docker compose restart

Error: "Environment variable not found"

  • Verifica que el archivo .env existe
  • Verifica que contiene todas las variables necesarias
  • Reinicia la terminal después de crear el archivo

Error al ejecutar seed.ts

  • Asegúrate de haber ejecutado npx prisma migrate dev primero
  • Verifica que el archivo productos.json existe
  • Verifica que PostgreSQL está corriendo
