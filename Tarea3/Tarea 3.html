Tarea 3: Base de Datos con Object Relational Mapping

En esta tarea empezaremos a utilizar la BD PostgreSQL usando un ORM: una capa de software encima de la BD que uniformiza su manejo, y con la que podríamos utilizar distintas DBMS en la fase de desarrollo y/o en la de producción sin cambiar el código. Además tienen herramientas para migraciones, inspeccionar la BD, etc. En cualquier caso siempre se podría usar directamente SQL si fuera necesario para optimizar el rendimiento, o por cualquier otra razón.

Instalación de la BD

Usaremos PostgreSQL en un contenedor docker, con docker compose. Usar la IA para crear el archivo docker-compose.yml, con imagen alpine, y los passwords con variables de entorno en un archivo aparte .env por motivos de seguridad. Poner en marcha la BD con doker compose up.

ORM

El manejo de la BD lo haremos con un el ORM Prisma. Seguimos mas o menos los pasos de la documentación en Quickstart. Esto es porque Prisma ha pasado recientemente de la versión 6 a la 7, con algunos problemas todavía.

De todos modos instalamos la última versión de prisma como en Getting Started with Prisma 7 with Nodejs & postgresql 

$ npm install -D prisma dotenv
$ npm install @prisma/client @prisma/adapter-pg
$ npx prisma init

Esta última orden modifica el archivo .env y crea una archivo prisma/schema.prisma. En el archivo .env ha puesto la conexión con una BD de desarrollo. En su lugar ponemos la conexión con nuestra BD en docker compose (Connection URL): 

DATABASE_URL="postgresql://yo:una_clave_muy_segura_123@localhost:5432/ssbw?schema=public"

Esquema de la BD

Comprobando en Overview of Prisma Schema, editamos el prisma/schema.prisma 

generator client {
  provider = "prisma-client"
  output   = "../generated/prisma"
  previewFeatures = ["fullTextSearchPostgres"]
}
	
datasource db {
  provider = "postgresql"
}
	
model Producto {
  id          Int     @id @default(autoincrement())
  título      String  @db.Char(127)
  descripción String  @db.Text
  precio      Decimal
  imagen      String  @db.Char(127)
}
Donde hemos activado la opción de busqueda en textos indexados de postgres Full-Text Search is now in preview for PostgreSQL. y puesto un modelo para la tabla de productos. Este modelo se encarga de validar los datos antes de que pasen a la BD, crear índices, mapear a tipos propios de la BD, etc.

A continuación creamos una migración para crear la(s) tablas 

$ npx prisma migrate dev --name esquema_inicial

y generamos el cliente para esta migración 

$ npx prisma generate

y lo configuramos en ./prisma/prisma.client.ts: 

import 'dotenv/config';
import { PrismaClient } from '../generated/prisma/client.ts'
import { PrismaPg } from '@prisma/adapter-pg'
			
// 1. Setup the adapter
const adapter = new PrismaPg({ connectionString: process.env.DATABASE_URL })
// 2. Initialize Prisma with the adapter
const prisma = new PrismaClient({ adapter })
			
// 3. export global prisma api
export default prisma

Los datos de la DB los podemos ver y modificar con el cliente gráfico 

$ npx prisma studio

Script para poner los datos iniciales en la BD

Haremos un script seed.ts para poner los datos de productos.json en la BD. 

import prisma from "./prisma/prisma.client.ts";

type Producto = {
  título: string
  descripción: string
  texto_precio: string
  imagen: string
}
type Productos = Producto[]

import productos from "./productos.json" with {type: 'json'}
await Guadar_en_DB(productos)

// Hacer alguna consulta para comprobar como:
// productos cuya descripción empieze por 'lámina' ordenados por precio
// etc

await prisma.$disconnect()

// async porque dentro hay await
async function Guadar_en_DB(productos: Productos): Promise<void> {

  for (const producto of productos) {
    const título = producto.título
    const descripción = producto.descripción
    const imagen = producto.imagen
    const precio = Number(producto.texto_precio.slice(0, -2).replace(/,/, '.'))
    //console.log(precio)
    try {
      const prod = await prisma.producto.create({
        data:{
          título,
          descripción,
          imagen,
          precio
        }
      })
      console.log('Creado', prod)
    } catch(error:any) {
      console.error(error.message)
    }
  } // end-for

} // end-function
